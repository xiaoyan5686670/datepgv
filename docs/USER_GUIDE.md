# 使用指南（USER GUIDE）

本指南面向业务人员 / 数据分析师，帮助你用自然语言生成 Hive / PostgreSQL / Oracle SQL，并理解会话与元数据管理功能。

## 1. 功能概览

系统提供：

- 自然语言 → SQL 生成（支持 Hive / PostgreSQL / Oracle）
- SQL 结构与引用表预览
- 会话记录管理（多轮追问 / 历史会话切换）
- 表元数据管理与多种导入方式（DDL、文件、CSV、数据库自动同步）

## 2. 首次配置

首次使用建议由技术同学先完成以下配置：

1. 打开前端首页：<http://localhost:3000>
2. 点击右上角 **“模型配置”** 进入 `/settings`
3. 选择或创建一个可用的 LLM 配置（如 OpenAI / Gemini）
4. 确保 Embedding 配置也已可用（用于表语义检索）

> 如果在对话中看到“缺少模型配置”或类似提示，请联系运维/开发人员到“模型配置”页面添加配置。

## 3. 导入表元数据

进入 **“元数据管理”** 页面：<http://localhost:3000/admin>

### 3.1 手动录入

- 通过页面表单，填写：
  - 数据库类型（Hive / PostgreSQL / Oracle）
  - 库名 / Schema（可选）
  - 表名、表注释
  - 列名、类型、注释、是否分区键
- 适合补充小量或零星的表结构。

### 3.2 DDL 文本导入

- 打开“DDL 解析导入”弹窗
- 在“文本粘贴”模式下：
  - 选择数据库类型（Hive / PostgreSQL / Oracle）
  - 粘贴一条或多条 `CREATE TABLE` 语句
  - 点击“解析并导入”

系统会自动解析 DDL 并生成多张表的元数据。

### 3.3 DDL 文件导入

- 在同一个“DDL 解析导入”弹窗中切换到 **“文件上传”** 模式
- 选择一个 `.sql / .ddl / .txt` 文件：
  - 支持多条 `CREATE TABLE` 语句
  - 支持常见编码（UTF-8 / GB18030 等）
- 选择数据库类型与数据库名（可选）
- 预览区会显示文件前若干行内容
- 点击“解析并导入”

若文件编码或语法存在问题，错误提示会指出：

- 文件编码问题
- 未发现 `CREATE TABLE`
- 方言与 DDL 不匹配等

### 3.4 CSV / Excel 批量导入

通过“CSV 批量导入”上传结构化文件：

必需列：

- `table_name`
- `column_name`
- `column_type`

可选列：

- `table_comment`
- `column_comment`
- `nullable`
- `is_partition_key`
- `tags`

示例：

```csv
table_name,column_name,column_type,table_comment,column_comment,nullable,is_partition_key,tags
ods_orders,order_id,STRING,订单表,订单ID,false,false,"ods,orders"
ods_orders,amount,DECIMAL,订单表,金额,false,false,
```

### 3.5 数据库自动同步（PostgreSQL）

如果你已有一个 PostgreSQL 业务库，可以通过 `/api/v1/metadata/sync/postgresql` 进行自动同步（需提供连接串 DSN 和 schema 名）。

## 4. 开始对话生成 SQL

打开首页：<http://localhost:3000>

### 4.1 选择 SQL 类型

在页面顶栏右侧可选择当前对话的目标 SQL 类型：

- Hive
- PostgreSQL
- Oracle

选择后，所有后续问题都会按照该类型来生成 SQL，并在结果中标记出来。

### 4.2 输入自然语言问题

在底部输入框中输入你的需求，例如：

- “查询过去 7 天每个部门的销售总额”
- “统计各状态订单数量及占比”
- “找出最近一个月新注册且有过购买记录的用户”
- “按城市汇总近 30 天的 GMV，取 Top 10”

按 Enter 发送，Shift+Enter 换行。

### 4.3 阅读返回结果

每一轮 AI 回复通常包括：

- **SQL 类型标签**：Hive SQL / PostgreSQL / Oracle SQL
- **引用表列表**：模型在生成 SQL 时认为相关的表
- **SQL 代码块**：完整的可执行 SQL，支持一键复制
- **流式状态**：正在检索元数据 / 正在生成 SQL 时显示旋转指示

如果后续对同一个问题有追问，系统会结合最近若干轮上下文生成新的 SQL。

## 5. 会话管理

### 5.1 会话列表

在桌面端左侧可以看到“会话”侧边栏：

- 顶部“新建会话”按钮：创建一个新的空会话。
- 下方列表展示最近的会话：
  - 标题：取自该会话首条用户问题（截断显示）。
  - 时间：最后一条消息的时间。
  - 点击某一项即可切换到对应历史会话，再继续追问。

### 5.2 清空对话 / 删除会话

两种方式：

- 在对话框右下角点击“清空对话”：
  - 删除当前会话记录，并创建一个全新的会话 ID。
- 在会话列表中点击某个会话右侧的“删除”图标：
  - 删除该会话及其所有历史记录。

> 删除只影响当前系统中的历史记录，不会影响实际业务数据库中的数据。

## 6. 常见问题（FAQ）

### 6.1 没有返回任何结果

- 检查是否已经在“元数据管理”中导入了相应业务表的元数据。
- 确认当前选择的 SQL 类型与表的 `db_type` 一致。

### 6.2 提示缺少 Embedding / LLM 配置

- 前往 `/settings` 页面，选择或创建一个有效配置。
- 确保 API Key 正确、模型名称与服务商一致。

### 6.3 DDL 文件导入失败

- 优先检查：
  - 文件扩展名是否为 `.sql/.ddl/.txt`
  - 文件编码是否为 UTF-8 / GB18030
  - 文件中是否包含合法的 `CREATE TABLE` 语句
- 参考错误提示中的具体原因进行修正。

### 6.4 SQL 不符合预期

- 检查表元数据是否完整（字段名 / 类型 / 注释）。
- 在提问中尽量给出时间范围、分组维度等关键信息。
- 可以通过再追问“把结果按部门排序”之类的指令进行微调。
